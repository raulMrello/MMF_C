Detailed Description

Tasks are pieces of software capable to be executed in a run-to-completion manner, this implies that they must return to allow other tasks to be executed (they never must block). Tasks can be STOPPED during initialization, WAITING for an event, READY for execution, RUNNING during execution and YIELD if allow other tasks to take control of the cpu.
